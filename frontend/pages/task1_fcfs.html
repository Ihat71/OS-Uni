What is FCFS Scheduling?

FCFS stands for First Come, First Served. It is one of the simplest CPU scheduling algorithms. In FCFS scheduling, the process that arrives first gets the CPU first. The CPU executes each process completely before moving to the next process.

Real-life example

FCFS is like students standing in a line. The first student in the line is served first, then the second, then the third, and so on. No one can skip the line.

Important points

  • FCFS is a non-preemptive scheduling algorithm.
  • Once a process starts, it keeps the CPU until it finishes.
  • It is simple and easy to understand.
  • It can cause long waiting time if a long process arrives first. This is called the convoy effect.

Advantages and Disadvantages

Advantages

  • Very simple to understand and implement.
  • Fair in the sense that processes are handled in the order they arrive.
  • No starvation, because every process eventually gets the CPU.

Disadvantages

  • Average waiting time can be high.
  • Short processes may wait behind long processes.
  • It is not suitable for time-sharing systems where fast response is important.
