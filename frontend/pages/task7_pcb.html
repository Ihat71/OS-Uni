What is a Process Control Block?

A Process Control Block, usually called a PCB, is a data structure stored by the operating system. It contains all the information needed to manage a process.

Think of the PCB as an official identity card and memory notebook for a process. When a process is running, waiting, interrupted, or terminated, the operating system uses the PCB to know exactly what the process is doing and what should happen next.

Process

A program in execution, such as a browser, editor, or calculator.

→

PCB

Stores the process state, ID, registers, program counter, memory data, and I/O details.

CPU

Runs one process at a time on a single core.

↔

Operating System

Uses PCBs to switch between processes safely and efficiently.

Main Information Stored in a PCB

Different operating systems may store slightly different details, but most PCBs contain the following important fields:

Process ID A unique number used to identify the process.
Process State Shows whether the process is new, ready, running, waiting, or terminated.
Program Counter Stores the address of the next instruction to execute.
CPU Registers Saves temporary CPU values so the process can continue later.
Memory Information Stores memory limits, page table data, or segment information.
Scheduling Information Includes priority, queue position, and CPU scheduling details.
I/O Status Tracks files, devices, and input/output operations used by the process.
Accounting Data Records CPU time used, owner, and other usage statistics.
Parent Process Stores information about the process that created this process.

Why the PCB is Important During Context Switching

A context switch happens when the CPU stops running one process and starts running another process. This may happen because of scheduling, interrupts, I/O waiting, or time quantum expiration.

During a context switch, the operating system saves the current process information into its PCB. Later, when the process is selected again, the operating system reloads the saved information from the PCB.

Context Switch Steps

  • Pause the currently running process.
  • Save CPU registers and program counter into its PCB.
  • Update the process state, such as Running → Ready or Waiting.
  • Load another process's PCB information into the CPU.
  • Resume execution from the saved program counter.

Real-Life Analogy

Imagine a teacher checking many students' exams. If the teacher pauses one exam and moves to another, they need a note showing where they stopped, what they already checked, and what remains. That note is like a PCB. It helps the operating system return to a process without losing progress.
