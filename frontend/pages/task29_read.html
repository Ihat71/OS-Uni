read() System Call

What is read() ?

The read() system call is used in Operating Systems to read data from a file, keyboard, pipe, or input stream into a buffer.

It is a low-level UNIX/Linux system call that communicates directly with the kernel.

Syntax: 

ssize_t read(int fd, void *buffer, size_t count);

Parameters:

  • fd → File descriptor
  • buffer → Memory location to store data
  • count → Number of bytes to read

How read() Works

Input Source

File / Keyboard / Pipe

➡

Kernel

Processes the request

➡

Buffer

Stores received data

C Language Example

#include <unistd.h>
#include <fcntl.h>

int main() {

    char buffer[100];

    int fd = open("file.txt", O_RDONLY);

    read(fd, buffer, 100);

    return 0;
}

In this example:

  • The file is opened in read-only mode.
  • read() copies data from the file into the buffer.
  • The operating system handles the transfer.

Return Values of read()

  • Positive Number → Number of bytes successfully read
  • 0 → End of file (EOF)
  • -1 → Error occurred

Why is read() Important?

The read() system call is one of the most important functions in UNIX/Linux operating systems because:

  • It enables communication with files and devices.
  • It is used internally by many programs.
  • It provides direct interaction with the operating system kernel.
  • It is essential for file handling and process communication.
