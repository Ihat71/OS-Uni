dup() and dup2()

This page explains how Linux/Unix systems duplicate file descriptors using dup() and dup2(). These system calls are important for file redirection, pipes, shells, and process I/O control.

Try Interactive Demo View C Example

// duplicate stdout to a file

int fd = open("output.txt", O_WRONLY | O_CREAT, 0644);

dup2(fd, STDOUT_FILENO);

printf("This goes to the file!");

// stdout now points to output.txt

1. What are dup() and dup2()?

dup()

dup() duplicates an existing file descriptor and returns the lowest available unused file descriptor number.

Example: if file descriptor 3 is open, dup(3) may return 4 if 4 is the lowest free descriptor.

dup2()

dup2() duplicates one file descriptor into a specific file descriptor number.

Example: dup2(fd, 1) makes file descriptor 1, which is standard output, point to the same file as fd.

Simple idea: A file descriptor is like a small number used by the operating system to refer to an open file, terminal, pipe, or socket. Duplicating a descriptor means creating another number that refers to the same open file/resource.

2. Standard File Descriptors

In Unix/Linux, every process normally starts with three standard file descriptors:

Descriptor Number Name C Constant Default Target
0 Standard Input STDIN_FILENO Keyboard / input stream
1 Standard Output STDOUT_FILENO Terminal screen / output stream
2 Standard Error STDERR_FILENO Terminal screen / error stream

3. Difference Between dup() and dup2()

dup(oldfd)

  • Duplicates oldfd.
  • Returns the lowest unused file descriptor.
  • You cannot choose the new descriptor number.
  • Useful when any free descriptor is acceptable.

dup2(oldfd, newfd)

  • Duplicates oldfd into newfd.
  • You choose the descriptor number.
  • If newfd is already open, it is closed first.
  • Very useful for input/output redirection.

5. C Program Example

This example redirects stdout to a file using dup2(). So instead of printing to the terminal, the text is written into output.txt. 

#include <stdio.h>
#include <unistd.h>
#include <fcntl.h>

int main() {
    int fd;

    // Open or create output.txt
    fd = open("output.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);

    if (fd < 0) {
        perror("open");
        return 1;
    }

    // Make stdout, file descriptor 1, point to output.txt
    dup2(fd, STDOUT_FILENO);

    // This will not appear on the terminal.
    // It will be written into output.txt.
    printf("Hello! This text was redirected using dup2().\n");

    close(fd);

    return 0;
}
Compile and run:
gcc dup_demo.c -o dup_demo
./dup_demo
cat output.txt

6. Why is this important?

Used in Shell Redirection

When you type ls > files.txt, the shell uses file descriptor redirection so the output of ls goes to a file instead of the terminal.

Used with Pipes

Commands like ls | grep .c depend on redirecting one process's output into another process's input. This is done using file descriptors.

7. Quick Check

Q1: Which function lets you choose the new file descriptor number?
✅ Answer: dup2()
Q2: What is the file descriptor number for standard output?
✅ Answer: 1, also called STDOUT_FILENO.
Q3: What does dup() return?
✅ Answer: It returns the lowest unused file descriptor.
