What is Shared Memory?

Shared Memory is a method of Interprocess Communication (IPC) where two or more processes share the same memory region. Instead of sending messages back and forth, processes directly read and write data in shared memory.


This communication method is very fast because data does not need to be copied multiple times. However, synchronization is required to prevent conflicts when multiple processes access the same data.

How the Demo Works

  1. Process A writes information into shared memory.
  2. The data becomes available in the shared memory region.
  3. Process B reads the same data from memory.
  4. The lock button simulates synchronization between processes.

Advantages and Disadvantages

Advantages

  • Very fast communication
  • Efficient for large amounts of data
  • Reduces CPU overhead
  • Widely used in operating systems

Disadvantages

  • Requires synchronization
  • Can cause race conditions
  • Harder to manage safely
  • Processes may overwrite data
