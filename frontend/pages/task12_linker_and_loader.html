What are the Linker and Loader?

In an operating system, a program does not run immediately after we write the source code. It must pass through several stages before it becomes a running process. Two important parts of this process are the linker and the loader.

Linker

The linker combines object files and required library files into one executable program. It connects functions, variables, and addresses that may be stored in different files.

Loader

The loader loads the executable program into main memory so the CPU can start executing it. It prepares the program as a process inside memory.

Simple Code Example

Suppose a program uses the printf() function. The source code has the function call, but the actual function exists in a library. The linker connects the program to that library function.

#include <stdio.h>

int main() {
    printf("Hello, Operating Systems!");
    return 0;
}

After linking, the executable is ready. Then the loader puts it into memory so it can run.

Types of Linking and Loading

Static Linking

Library code is included inside the executable file before the program runs. The executable may become larger, but it can run without depending on external library files.

Dynamic Linking

Library code is linked while the program is running. This saves memory because many programs can share the same library.

Absolute Loading

The program is loaded into a fixed memory location.

Relocatable Loading

The program can be loaded into different memory locations. This gives the operating system more flexibility.

Why Linker and Loader are Important

  • They help convert a program into a form that can actually run.
  • The linker connects different parts of a program together.
  • The loader places the executable program into memory.
  • They make program execution possible inside the operating system.
