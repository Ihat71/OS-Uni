Theory and Lab Concept

Shortest Job First (SJF) is a CPU scheduling algorithm that selects the process with the smallest burst time next. In this lab-style demonstration, the scheduler decides the execution order based on each process burst length, minimizing average waiting time when arrival times are equal or when all processes are ready.

There are two main SJF variants:

  • Non-preemptive SJF: Once the CPU starts running a process, it runs until completion before the scheduler picks the next shortest job.
  • Preemptive SJF (also called Shortest Remaining Time First): The scheduler may interrupt a running process if a newly arrived process has a shorter remaining burst time.

This page demonstrates the non-preemptive version, which is commonly used in teaching labs because it makes the order and waiting time calculations easy to follow.

Key terms

  • Burst time: The time required for a process to execute on the CPU.
  • Arrival time: The time at which a process becomes ready for execution.
  • Waiting time: How long a process waits in the ready queue before it begins execution.
  • Turnaround time: Total time from arrival to completion of a process.
Lab concept: SJF aims to reduce the average waiting time in the ready queue by always choosing the shortest next process. This often improves throughput, but may cause starvation if long processes are repeatedly delayed.
