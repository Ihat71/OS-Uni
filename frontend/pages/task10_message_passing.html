// Inter-Process Communication //

MESSAGE PASSING MODEL

Processes communicate by sending & receiving messages — no shared memory needed

Core Concepts
📨

Message Passing

Processes exchange data by explicitly sending and receiving messages via the OS kernel — safe across machines.

📬

Mailbox / Port

An indirect communication endpoint. Multiple processes can send to the same mailbox; OS queues the messages.

🔄

Synchronous

Blocking: sender waits until receiver gets the message. Both processes rendezvous at the same time.

⚡

Asynchronous

Non-blocking: sender drops message in buffer and continues. Receiver picks it up later independently.

Direct Communication

  • Sender names the receiver explicitly: send(P, msg)
  • Receiver may name the sender: receive(Q, &msg)
  • One link per pair of communicating processes
  • Tight coupling — changing a name breaks the link
  • Example: Unix pipes between parent/child processes

Indirect Communication

  • Messages sent to/from mailboxes (ports)
  • send(A, msg) — send to mailbox A
  • receive(A, &msg) — receive from mailbox A
  • Multiple processes can share a mailbox
  • Example: POSIX message queues, Windows mailslots

send( ) Semantics

  • Blocking send: waits until message is received
  • Non-blocking send: puts message in buffer, returns immediately
  • OS kernel copies data from sender address space → message buffer
  • Returns error if queue is full (in bounded buffer mode)

receive( ) Semantics

  • Blocking receive: waits until a message arrives
  • Non-blocking receive: returns NULL if no message
  • OS kernel copies data from message buffer → receiver address space
  • Message is removed from queue after delivery
