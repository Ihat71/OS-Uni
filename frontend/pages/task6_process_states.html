Process State Explanation

New State

The New state is the birth stage of a process. When you initiate a program, the operating system's long-term scheduler picks it up and begins allocating the necessary primary memory (RAM) and resources it needs to run.
At this point, the process control block (PCB)—which tracks the process's ID, priority, and register data—is created. Crucially, the process is not yet ready to execute; it is simply being structured and loaded into memory from secondary storage.

Ready State

Once the operating system finishes setting up the process, it moves into the Ready state. Here, the process is fully loaded into main memory, has all its required resources, and is actively waiting for its turn to use the CPU.
Processes in this state sit in a "ready queue." The short-term scheduler (or CPU dispatcher) constantly monitors this queue, picking the next process to hand over to the processor based on scheduling algorithms like Round-Robin or First-Come-First-Served.

Running State

The Running state is where the actual work happens. When the CPU scheduler selects a process from the ready queue, it enters this state and its instructions begin executing on the processor core.
A process can only stay here until one of three things happens: it finishes its task, it gets interrupted by the OS because its time slice (quantum) ran out, or it requests an external resource (like reading a file) that forces it to pause.

Waiting State

Also known as the Blocked state, a process enters Waiting if it cannot continue running until a specific event occurs. This usually happens when the process requests an Input/Output (I/O) operation—like waiting for a user keystroke, fetching data from a hard drive, or downloading a file over the network.
To prevent the CPU from sitting idle, the OS moves this blocked process out of the processor and into a waiting queue, freeing up the CPU to run other processes. Once the event finishes, the process goes back to the Ready state, not directly back to Running.

Terminated State

The Terminated state is the final resting place of a process. A process arrives here either because it successfully finished executing all its code or because it was forcefully killed by the OS due to an error (like a segmentation fault or running out of memory).
In this stage, the operating system performs cleanup: it reclaims the RAM, closes any open files, deletes the Process Control Block (PCB), and frees up all resources back to the system pool.

EXPLANATION

This simulator demonstrates how operating systems manage processes using different process states and CPU scheduling techniques.

STEP 1 — CREATE A PROCESS

The user first enters a process name and burst time.

  • Process Name: Identifies the process such as P1, P2, Chrome, or Game.
  • Burst Time: Represents the total CPU execution time required by the process.

When the "Create Process" button is pressed, a new process object being placed inside the NEW state.

STEP 2 — PROCESS MOVES TO READY STATE

After initialization, the process automatically moves from NEW state to READY state.

READY state represents processes waiting for CPU allocation. The process cannot execute immediately if CPU cores are busy.

STEP 3 — SELECT CPU MODE

The CPU Mode dropdown allows the user to choose the number of CPU cores.

  • Single Core = 1 running process at a time
  • Dual Core = 2 running processes simultaneously
  • Quad Core = 4 running processes simultaneously

Increasing CPU cores improves multitasking and allows multiple processes to execute together.

STEP 4 — START SIMULATION

When the "Start Simulation" button is pressed, the CPU scheduler starts assigning READY processes to available CPU cores.

The scheduler continuously checks:

  • Available CPU cores
  • Processes inside READY queue
  • Remaining burst times
  • Waiting processes

STEP 5 — RUNNING STATE

Processes assigned to CPU cores enter RUNNING state.

During execution:

  • Remaining time decreases every second
  • The process executes CPU instructions
  • The process continues until completion

The RUNNING section visually displays processes currently being executed by the CPU.

STEP 6 — WAITING STATE

Some processes randomly move into WAITING state to simulate I/O operations.
(Simulator automaticly acts like its having I/O operation and takes the process to waiting state.)

Examples of I/O operations:

  • Disk access
  • Keyboard input
  • Internet communication
  • File loading

After waiting finishes, the process returns back to READY queue.

STEP 7 — TERMINATED STATE

When remaining burst time reaches zero, the process completes execution and enters TERMINATED state.

This means the operating system removes the process from CPU execution.

PROCESS CALCULATION TABLE

The calculation table tracks process information in real time.

  • Process Name
  • Burst Time
  • Remaining Time
  • Current Process State

The table updates dynamically during execution.

SIMULATION LOG SECTION

The simulation log records every process event and transition.

Example logs:

  • P1 entered READY queue
  • P1 assigned to CPU Core 1
  • P1 moved to WAITING state
  • P1 returned to READY queue
  • P1 terminated successfully

RESET BUTTON

The Reset button clears:

  • All processes
  • CPU core assignments
  • Logs
  • Tables
  • Process states

This allows the simulation to restart from the beginning.
