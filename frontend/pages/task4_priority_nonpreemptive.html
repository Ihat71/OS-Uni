CPU Scheduling

The CPU can only run one process at a time. When multiple processes are ready, the OS needs a rule for who goes first — that rule is the scheduling algorithm.

Priority Scheduling

Each process gets a priority number. When the CPU is free, it picks the process with the lowest priority number (highest urgency) from the ready queue. Ties go to whoever arrived first.

Non-Preemptive

Once a process starts running it goes all the way to completion — even if a higher-priority process arrives mid-execution. That new arrival simply waits in the queue.

Priority 1 = Highest No interruptions
