Linux System Call

sleep()

Pause process execution for a specified number of seconds — one of the simplest yet most essential system calls in Unix/Linux.

↓
01 — Overview

What is sleep()?

The sleep() function causes the calling process (or thread) to be suspended from execution for a given number of seconds. During this time, the process transitions from the Running state to the Sleeping (Waiting) state. The operating system's scheduler removes it from the CPU and gives that time to other processes.

Once the specified duration elapses, the kernel moves the process back to the Ready queue, and it resumes execution when the scheduler next picks it up.

It is defined in <unistd.h> and its signature is:

C #include <unistd.h>

unsigned int sleep(unsigned int seconds);

// Returns: 0 if the full time elapsed,
// or the number of unslept seconds if interrupted by a signal.
02 — Mechanism

How Does It Work Internally?

When a process calls sleep(n), the following happens inside the operating system:

① Trap to Kernel

The process issues a system call, switching from User Mode to Kernel Mode via a software interrupt.

② Set Timer

The kernel sets up an internal timer (often using nanosleep() underneath) for the requested duration.

③ Block Process

The process is moved to the Waiting state. The scheduler picks another ready process to run on the CPU.

④ Timer Expires

When the timer fires, the kernel moves the process back to the Ready queue. It resumes when scheduled.

04 — Code Example

A Simple C Program Using sleep()

example.c #include <stdio.h>
#include <unistd.h>

int main() {
    printf("Starting... PID = %d\n", getpid());

    for (int i = 1; i <= 3; i++) {
        printf("Doing work... iteration %d\n", i);
        sleep(2);  // pause for 2 seconds
    }

    printf("Done!\n");
    return 0;
}

Compile and run with gcc example.c -o example && ./example. You will see a 2-second pause between each iteration as the process sleeps.

06 — Related Functions

Variants & Alternatives

nanosleep() Nanoseconds <time.h> High precision; preferred for fine-grained delays.
alarm() Seconds <unistd.h> Schedules a SIGALRM signal instead of blocking.
07 — Key Takeaways

Things to Remember

Non-busy Wait

sleep() does not spin-loop on the CPU. The process is truly removed from the run queue, freeing the CPU for other work.

Signal Interruption

A signal (e.g., SIGINT) can wake the process early. The return value tells you how many seconds were left unslept.

Minimum Guarantee

The actual sleep time may be slightly longer than requested due to scheduling latency, but never shorter (unless interrupted).
