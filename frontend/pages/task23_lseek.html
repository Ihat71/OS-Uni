Task 23: The lseek() System Call

Concept Explanation

In Operating Systems, every open file has an associated "file offset," which dictates the current byte position where the next read() or write() operation will occur. The lseek() system call allows a process to explicitly reposition this read/write file offset.

Syntax

off_t lseek(int fd, off_t offset, int whence);

Parameters

  • fd: The file descriptor of the open file.
  • offset: The number of bytes to move the pointer. This can be positive or negative.
  • whence: The reference point for the offset. It takes one of three constants:
    • SEEK_SET: The offset is set to offset bytes from the very beginning of the file.
    • SEEK_CUR: The offset is set to its current location plus offset bytes.
    • SEEK_END: The offset is set to the size of the file plus offset bytes.

Note: Moving the pointer past the end of the file and writing data creates a "hole" (a sparse file) where unwritten bytes read as zeros.
